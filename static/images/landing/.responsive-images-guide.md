# Responsive Images Setup

This directory contains optimized images for the landing page.

## Current Images

- **og-image.webp** (1200×630px) - OpenGraph preview, 8.6K
- **og-image.png** (1200×630px) - PNG fallback, 35K
- **twitter-image.webp** (1200×675px) - Twitter card, 8.8K
- **twitter-image.png** (1200×675px) - PNG fallback, 34K

## Pattern for Future Responsive Images

All future images should follow these patterns:

### Tech Logo Images (48×48px)
```html
<picture>
  <source srcset="tech-logo-1x.webp 1x, tech-logo-2x.webp 2x" type="image/webp">
  <source srcset="tech-logo-1x.png 1x, tech-logo-2x.png 2x" type="image/png">
  <img src="tech-logo-1x.png" alt="Technology name" loading="lazy" width="48" height="48">
</picture>
```

### Feature Screenshots (Responsive Width)
```html
<picture>
  <source srcset="feature-480w.webp 480w, feature-768w.webp 768w, feature-1440w.webp 1440w" type="image/webp">
  <source srcset="feature-480w.png 480w, feature-768w.png 768w, feature-1440w.png 1440w" type="image/png">
  <img src="feature-480w.png" 
       srcset="feature-480w.png 480w, feature-768w.png 768w, feature-1440w.png 1440w"
       sizes="(max-width: 768px) 90vw, (max-width: 1440px) 60vw, 800px"
       alt="Feature description" loading="lazy" width="800" height="450">
</picture>
```

## Naming Convention

- **1x variants** - Standard density (48px, 100px, etc.)
- **2x variants** - Retina/high-DPI (96px, 200px, etc.)
- **Width variants** - Responsive width (480w, 768w, 1440w)
- **Formats** - `.webp` (primary) and `.png` (fallback)

## Optimization Guidelines

- **Target file size**: <50KB per image
- **Quality settings**:
  - WebP: 85 quality (lossy)
  - PNG: 9 compression (lossless)
- **Lazy-loading**: All images except above-fold hero should have `loading="lazy"`
- **Dimensions**: Always include `width` and `height` to prevent CLS (Cumulative Layout Shift)

## Conversion Commands

```bash
# PNG to WebP (85% quality)
convert input.png -quality 85 output.webp

# Create 2x version
convert input.png -resize 200% output-2x.png
convert output-2x.png -quality 85 output-2x.webp

# Optimize PNG (9 compression)
convert input.png -quality 9 output-optimized.png
```
